# 需求 4：关键字搜索

## 动机

列表页数据增多后，运营人员需要快速定位目标配置。当前列表无搜索功能。

## 优先级

低。数据量小（每类 < 100），但搜索是基本的用户体验。

## 预期效果

每个列表页顶部有搜索框，输入关键字后实时过滤，匹配 name 或 config 中的字段值。

## 改动范围

2 个文件：GenericList.vue + NpcTemplateList.vue。纯前端过滤，不改后端。

## 验收标准

- **R1**：GenericList 搜索框输入关键字，表格按 name 和 config 摘要过滤
- **R2**：NpcTemplateList 搜索框输入关键字，按 name / preset / 组件名过滤
- **R3**：搜索为空时显示全部数据
- **R4**：`npm run build` 通过

## 不做什么

- ❌ 不做后端搜索 API（数据量 < 100，前端过滤足够）
- ❌ 不做模糊拼音搜索
