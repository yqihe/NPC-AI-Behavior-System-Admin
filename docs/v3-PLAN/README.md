## 侧边栏结构

```
配置管理（可折叠）
  ├─ 字段管理
  ├─ 模板管理
  └─ NPC 管理

行为管理（可折叠）
  ├─ 事件类型
  ├─ 状态机
  └─ 行为树

世界管理（可折叠）
  └─ 区域管理

系统设置（可折叠）
  ├─ Schema 管理
  └─ 导出管理
```

## 9 个页面

| # | 页面 | 分类 | 一句话 |
|---|------|------|--------|
| 1 | 字段管理 | 配置管理 | 定义 NPC 可以有什么属性（血量、等级、性格...），是最底层原子单位 |
| 2 | 模板管理 | 配置管理 | 把字段组合成可复用的模板（战斗生物、环境NPC...） |
| 3 | NPC 管理 | 配置管理 | 选模板，填值，创建具体的 NPC 角色 |
| 4 | 事件类型 | 行为管理 | 定义游戏世界中会发生什么事（爆炸、枪声...） |
| 5 | 状态机 | 行为管理 | 定义 NPC 有哪些状态，什么条件下切换 |
| 6 | 行为树 | 行为管理 | 定义 NPC 在每个状态下具体做什么 |
| 7 | 区域管理 | 世界管理 | 定义游戏场景、天气、NPC 刷怪规则 |
| 8 | Schema 管理 | 系统设置 | 查看系统预定义的数据结构（开发者工具） |
| 9 | 导出管理 | 系统设置 | 导出配置给游戏服务端 |

## 推荐创建顺序

```
字段(1) → 模板(2) → 事件类型(4) → 状态机(5) → 行为树(6) → NPC(3) → 区域(7)
```

## 通用功能（所有页面适用）

### 搜索与分页
1. 所有列表页后端分页（20 条/页），不做前端全量过滤
2. 组合筛选走后端 MySQL 查询
3. 下拉筛选选项从数据库动态获取，不硬编码

### 用户体验
4. Dashboard 步骤引导，每个页面顶部操作说明
5. 可折叠侧边栏（el-sub-menu）
6. 所有枚举选项有中文标签
7. 所有表单字段有中文标题 + 描述提示
8. 操作反馈：成功/失败提示，loading 防重复，删除确认

### 数据架构
9. MongoDB 是唯一数据源，MySQL 是搜索索引
10. 写入流程：MongoDB → MQ → MySQL（异步同步）
11. 字段标识命名约束：`^[a-z][a-z0-9_]*$`
12. 所有实体软删除名称仍占用唯一性
