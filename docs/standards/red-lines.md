# 通用禁止红线

适用于所有项目。语言/框架专属禁令见 `go-red-lines.md`、`frontend-red-lines.md`。项目架构专属禁令见 `../architecture/red-lines.md`。

## 禁止静默降级

- **禁止**对无效配置值做静默降级（fallback to default）而不记录日志。配置错误必须在加载阶段报错或至少输出 WARN 日志
- **禁止**在运行时吞掉"不影响启动但影响行为"的错误。典型：条件 `op` 填错导致 `Evaluate()` 永远返回 false，系统正常运行但行为静默失效——比 crash 更难发现
- **禁止**在 lookup 失败时 silent return/continue。找不到注册类型、配置项、映射 key 时，必须记日志（至少 Debug 级别）
- **禁止**引用检查中 `json.Unmarshal` 失败时 `continue` 跳过。数据损坏应阻止操作，不能静默放行导致误删被引用的配置

## 禁止安全隐患

- **禁止**将外部输入直接拼接为文件路径或数据库查询。防止目录穿越和操作符注入
- **禁止**数据库查询中接受用户输入的 `$` 开头 key。查询条件必须用精确匹配
- **禁止**配置错误延迟到运行时暴露。引用关系必须在加载/保存阶段校验
- **禁止**使用无超时的外部 IO 调用。所有网络请求、数据库操作必须带 `timeout`
- **禁止**信任前端校验。所有校验必须在后端重复执行
- **禁止**在 API 响应中暴露内部错误信息或数据库连接串

## 禁止测试质量低下

- **禁止**测试中硬编码依赖外部数据的精确计数。用 `> 0` 或检查具体属性
- **禁止**测试中忽略反序列化/解析错误
- **禁止**测试中依赖外部文件路径或特定环境。测试数据内联或用 `runtime.Caller` 定位项目根目录
- **禁止**测试中存在未使用的变量或空断言（`if condition { /* 什么都不做 */ }`）
- **禁止**测试中存在死代码（未调用的 helper、被注释的测试）

## 禁止过度设计

- **禁止**引入没有使用场景的依赖或框架
- **禁止**添加没有对应实现的配置字段或环境变量
- **禁止**为只有一个调用点的逻辑创建抽象层

## 禁止协作失序

- **禁止**收到姐妹项目的需求/方案/联调请求后，不先回复就开始编码
- **禁止**口头答应但不落文档。架构变更一旦确认，必须在文档中体现
- **禁止**配置变更只修改本地参考文件（如 `configs/`）不通过 API 写入数据库。本地文件是参考，数据库才是真实数据源
