# 缓存模式禁止红线

适用于所有使用缓存的项目。描述的是缓存模式（Cache-Aside 等），不针对具体中间件（Redis 专属见 `redis-red-lines.md`）。

## 禁止缓存与数据库不一致

- **禁止**写操作成功后不清除缓存。必须先写 DB，成功后删缓存（Cache-Aside）
- **禁止**缓存 DELETE 失败时静默吞错。必须记日志告警
- **禁止**缓存无 TTL。所有 key 必须设置过期时间，防止永久脏数据

## 禁止缓存失效遗漏

- **禁止**批量修改操作只清列表缓存不清详情缓存。凡是修改了 detail 中包含的字段，都要同时清理 detail 缓存

## 禁止缓存击穿不设防

- **禁止**热 key 缓存过期时不做并发控制。高并发下缓存过期瞬间所有请求穿透到 DB 造成击穿。必须使用分布式锁（`TryLock` + double-check）或 singleflight 模式，获取锁失败时降级直查 DB 而非阻塞

## 禁止 TOCTOU 遗漏

- **禁止**批量操作的事务内不做二次检查。单条删除有 `FOR SHARE` 防 TOCTOU，批量删除也必须在每个字段的事务内重新检查。"事务外预检 + 事务内复查" 是标准模式，不能省略事务内复查

## 禁止缓存性能陷阱

- **禁止**在高频调用路径中做 `sort.Slice`。预排序缓存结果，调用时零分配返回
